# Content Index

## Purpose

This index tracks the written guidance that supports the diagram library.
It exists so the visual repo can be maintained like a real content system, not just a folder of exports.

## Sections

- diagram family notes
- export guidance
- source file guidance
- publication guidance
- naming and indexing guidance
- cross-repository linking guidance

## Use

- keep written guidance close to the visuals
- update the index when a new diagram family is added
- document any new export format or file convention here first
