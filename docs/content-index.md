# Content Index

## Purpose

This index organizes the diagram library into a front door for architecture, governance, reliability, resilience, and FinOps visuals.
Use it to quickly find the right visual family for a paper, a web page, or a board-facing artifact.

## Diagram Families

- [MCGR](../mcgr/README.md): ecosystem map, framework model, repo relationships
- [SRE](../sre/README.md): incident response, error budgets, reliability loops
- [Observability](../observability/README.md): telemetry, signals, anomaly detection
- [Disaster Recovery](../disaster-recovery/README.md): failover, failback, recovery governance
- [FinOps](../finops/README.md): allocation, optimization, executive cost reporting
- [Governance](../governance/README.md): policy flow, control ownership, review boards
- [AI Governance](../ai-governance/README.md): model risk, review flow, responsible AI
- [Enterprise Architecture](../enterprise-architecture/README.md): blueprints, capabilities, roadmaps

## Supporting Assets

- [Source Files](../source-files/README.md): editable originals
- [Exports](../exports/README.md): publication-ready outputs
- [Bibliography](../references/bibliography.md): shared references and source themes

## Index Rules

- keep one family page per diagram topic
- add a short purpose statement to every family page
- use the same naming pattern for exported files
- keep the index current whenever a new diagram is added
- prefer SVG for web and PNG/PDF for presentation or publication use

## Shared Direction

Use the same section structure as the other core repos so visual assets stay easy to find and reuse.
Keep family names and export names aligned so the same visual can be found across repos quickly.
